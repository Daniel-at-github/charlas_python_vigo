¿Qué es loguear?

Registrar acciones y estado de una aplicación en un soporte secundario

  • Salida estándar
  • Fichero
  • Socket
  • ...

¿Por qué loguear?

  • Diagnosticos: Cuando sucede un error podemos localizar en el código los logs previos al error para obtener contexto

  • Auditoría: Analizar los datos generados para generar información útil de nuestro negocio

¿Por qué usar logging y no prints?

print = texto por salida estándar

log = texto sensible al contexto

Algunas ventajas de usar logging

  • Categorización

  • Separación de responsabilidades: Usuarios con distinto nivel de responsabilidades pueden prescindir de los logs que no les interesen

Ej. El programador decide qué lugares del código pueden generar un log y con qué nivel. El sysadmin decide qué nivel de logs le interesa que se lleven a cabo.

Uso básico

import logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger()
logger.warning('WARNING >= INFO por tanto este mensaje se mostrará')
logger.debug('DEBUG < INFO por tanto este mensaje NO se mostrará')
# Esta segunda ejecución no tendrá ningún efecto
logging.basicConfig(level=logging.CRITICAL)

NOTA

basicConfig sólo funciona la primera vez que se llama durante la ejecución.

Nombre Descripción Ejemplo
filename Usa FileHandler con la ruta de fichero "/var/logs/logs.txt"
format Formato de la cadena "'%(asctime)s %(message)s'"
datefmt Formato de fecha "%H:%M:%S"
level Nivel del logger raíz "INFO"

Uso avanzado

config = {
    'disable_existing_loggers': False,  # Avoid dictConfig to disable previously instantiated global loggers
    'version': 1,
    'formatters': {
        'short': {
            'format': '%(asctime)s %(levelname)s %(name)s: %(message)s'
        },
    },
    'handlers': {
        'console': {
            'level': 'INFO',
            'formatter': 'short',
            'class': 'logging.StreamHandler',
            'filters': ['MyFilter', 'ContextFilter']
        },
        'file': {
            'level': 'INFO',
            'class': 'logging.FileHandler'
        }
    },
    'loggers': {
        '': {
            'handlers': ['console'],
            'level': 'ERROR',
        },
        'plugins': {
            'handlers': ['file'],
            'level': 'INFO',
            'propagate': False
        },
        'sistema': {
            'handlers': ['console'],
            'level': 'INFO',
        }

    },
}
import logging.config
logging.config.dictConfig(config)

Por partes...

LOGGER

Objeto a través del cuál se trata de emitir un mensaje categorizado

my_logger = logging.getLogger()
my_logger.critical('Todo ha fallado. Entremos en pánico')

LOG RECORD

  • Cada vez que un Logger trata de emitir un mensaje, se genera un objeto Log Record

  • Los Log Records además del mensaje y el nivel contienen datos de contexto (fecha, módulo, proceso, hilo, ...)

HANDLER

Recibe un Log Record y lo procesa a alguna salida según sus instrucciones

  • FileHandler
  • TimedRotatingFileHandler
  • StreamHandler
  • SMTPHandler
  • SocketHandler
  • HTTPHandler

FORMATTER

Recibe un Log Record y lo serializa como cadena de texto en base al esquema que tenga definido

'%(asctime)s %(levelname)s %(name)s: %(message)s'
2017-09-21 18:42:11,625 ERROR testing.loggers: Error al procesar...

https://docs.python.org/3/library/logging.html#logrecord-attributes

FILTER

Recibe un Log Record y puede hacer dos cosas:

  • Decidir si cumple las condiciones para ser emitido

  • Modificarlo para añadirle información extra

Se pueden aplicar múltiples filtros tanto a Loggers como a Handlers, teniendo que cumplirse las condiciones de todos ellos para que el mensaje se emita finalmente

config = {
    'disable_existing_loggers': False,  # Avoid dictConfig to disable previously instantiated global loggers
    'version': 1,
    'formatters': {
        'short': {
            'format': '%(asctime)s %(levelname)s %(name)s: %(message)s'
        },
    },
    'handlers': {
        'console': {
            'level': 'INFO',
            'formatter': 'short',
            'class': 'logging.StreamHandler',
            'filters': ['MyFilter', 'ContextFilter']
        },
        'file': {
            'level': 'INFO',
            'class': 'logging.FileHandler'
        }
    },
    'loggers': {
        '': {
            'handlers': ['console'],
            'level': 'ERROR',
        },
        'plugins': {
            'handlers': ['file'],
            'level': 'INFO',
            'propagate': False
        },
        'sistema': {
            'handlers': ['console'],
            'level': 'INFO',
        }

    },
}
import logging.config
logging.config.dictConfig(config)

Apunte final

Jerarquía de Loggers

Si al obtener un logger, lo hacemos con dot notation

my_logger = logging.getLogger('plugins.sistema')

El Log Record se procesará recursivamente por los handler que correspondan, hasta que se llegue o al raíz o a alguno que tenga "propagate: False"

En este ejemplo, se procesará por el handler 'sistema' y después por el handler 'plugins'

Recordemos que la variable global __name__ en python suele guardar una cadena con la jerarquía de módulos separados por punto...😍

# __name__ = projectA.moduleB.moduleC...
my_logger = logging.getLogger(__name__)
